## Incident Overview

**Incident ID:** INC-004  
**Title:** Suspicious Scheduled Task Activity Detected  
**Severity:** Medium  
**Status:** Resolved  
**Date:** July 8, 2026  

---

## Environment

- **Domain:** LAB.local
- **Affected Host:** DC01
- **Operating System:** Windows Server 2022
- **Monitoring Platform:** Wazuh
- **Endpoint Visibility:** Sysmon
- **Detection Source:** Sysmon Event Logs forwarded through Wazuh

---

# Description

A scheduled task activity was simulated on the domain controller to test detection and monitoring capabilities.

Scheduled tasks are commonly abused by attackers to maintain persistence after gaining access to a system. Attackers may create scheduled tasks that automatically execute malicious programs during startup, login, or at specific time intervals.

This simulation generated Sysmon telemetry related to Windows Task Scheduler activity, which was successfully collected and analyzed using Wazuh.

---

| Technique | Name | Description |
|---|---|---|
| T1053 | Scheduled Task/Job | Attackers may use scheduled tasks to establish persistence or execute malicious commands automatically |

# Analyst Findings

The detected activity matched scheduled task behavior associated with MITRE ATT&CK technique T1053.

The investigation determined:

- The executable was located in the expected Windows system directory.
- The binary was digitally signed by Microsoft.
- The activity was performed using an authorized administrator account.
- No suspicious payload execution was observed.
- No indicators of compromise were identified.

---

# Response Actions

1. Reviewed Wazuh alerts generated from Sysmon telemetry.
2. Investigated the process path and executable signature.
3. Verified the user account responsible for the activity.
4. Determined the activity was benign simulated administrative behavior.
5. Documented detection and investigation procedures.

---

# Lessons Learned

This investigation demonstrated:

- Successful collection of Sysmon telemetry through Wazuh.
- Detection of MITRE ATT&CK technique T1053.
- Importance of validating process location and digital signatures.
- The need for analysts to investigate alerts before classifying them as malicious.

**Classification:** Benign / Authorized Activity
